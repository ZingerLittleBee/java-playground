# 工厂模式

## 简单工厂模式
### 使用场景
- 需要创建的对象较少
- 客户端不关心对象的创建过程

### 优缺点
- 优点: 可以对创建的对象进行"加工"， 对客户端端隐藏相关细节
- 缺点: 
    - 因创建逻辑复杂或创建对象过多而造成代码臃肿
    - 新增、删除子类均会违反开闭原则

## 工厂方法模式
简单工厂模式的进一步抽象和拓展

### 优缺点
- 优点: 
  - 遵循开闭原则
  - 对客户端隐藏对象的创建细节
  - 遵循单一职责
- 缺点:
  - 添加子类的时候, 还需要添加与之对应的工厂类
  - 只支持同一类产品的创建


## 抽象工厂模式
定义: 提供一个创建一系列相关或相互依赖对象的接口
- 抽象工厂模式侧重的是统一产品足
- 工厂方法模式更加侧重于同一产品等级

解决了工厂模式只支持生产一种产品的弊端
